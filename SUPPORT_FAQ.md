# ❓ الأسئلة الشائعة - نظام خدمة العملاء

## 1️⃣ **المستخدم يرسل الشكوى - ماذا يحدث؟**

### السؤال:
بعد ما المستخدم يرسل الشكوى، الرسالة تروح فين؟

### الإجابة:
الشكوى تمر برحلة:

1. **المستخدم يرسل** → البيانات تذهب إلى الـ Backend
2. **الـ Backend يحفظ** → في قاعدة البيانات في جدول `support_tickets`
3. **الحالة الابتدائية** → `OPEN` (مفتوحة)
4. **الـ Frontend يُظهر** → رسالة نجاح للمستخدم
5. **النظام يُنتظر** → حتى يرد المسؤول

---

## 2️⃣ **كيف أشوف الشكاوى كمسؤول؟**

### الخطوات:
1. سجل دخول بحساب **ADMIN**
2. في الـ Navbar العلوي، ستجد زر جديد:
   - 🎧 **إدارة الدعم** (للمسؤولين فقط)
3. اضغط على الزر
4. تظهر لوحة التحكم الكاملة

### ماذا ترى؟
- 📊 إحصائيات (عدد الشكاوى المفتوحة، المغلقة، إلخ)
- 📋 قائمة جميع الشكاوى
- 🔍 بحث وفلترة
- 💬 تفاصيل كل شكوى

---

## 3️⃣ **أين تُحفظ الشكاوى؟**

### الإجابة:
في **قاعدة البيانات PostgreSQL** في جدول:
```
support_tickets
```

### المجلد والملفات:
- 📁 `server/routes/supportRoutes.js` - الـ API
- 📁 `server/prisma/schema.prisma` - الـ Database Schema

### البيانات المحفوظة:
```json
{
  "id": "معرف فريد",
  "userId": "معرف المستخدم (إذا كان مسجل)",
  "name": "اسم المرسل",
  "email": "البريد الإلكتروني",
  "subject": "الموضوع",
  "category": "TECHNICAL/ACCOUNT/PAYMENT/...",
  "message": "الرسالة",
  "priority": "LOW/NORMAL/HIGH/URGENT",
  "status": "OPEN/IN_PROGRESS/REPLIED/CLOSED",
  "replies": [],
  "createdAt": "تاريخ الإرسال",
  "updatedAt": "تاريخ آخر تحديث"
}
```

---

## 4️⃣ **كيف أرد على الشكوى؟**

### الخطوات:
1. افتح لوحة التحكم (زر الإدارة)
2. اختر شكوى من القائمة (اليسار)
3. في صفحة التفاصيل (اليمين):
   - اقرأ الرسالة الأصلية
   - في الأسفل، يوجد صندوق "اكتب ردك هنا..."
4. اكتب الرد
5. اضغط زر **إرسال**
6. رسالة نجاح تظهر: "تم إرسال الرد بنجاح ✅"

### ماذا يحدث في الخلفية؟
- الرد يُضاف إلى مصفوفة `replies`
- الحالة تتغير تلقائياً من `OPEN` إلى `REPLIED`
- تاريخ التحديث يُحفّظ (`updatedAt`)

---

## 5️⃣ **كيف أغلق الشكوى؟**

### الخطوات:
1. في صفحة تفاصيل الشكوى
2. في الأعلى اليمين، يوجد dropdown بالحالات:
   ```
   [الحالة: REPLIED ▼]
   ```
3. اختر **CLOSED**
4. يتم الحفظ تلقائياً

### الحالات المتاحة:
- 🔵 `OPEN` - مفتوحة (جديدة)
- 🟡 `IN_PROGRESS` - قيد المعالجة
- 🟢 `REPLIED` - تم الرد
- ⚪ `CLOSED` - مغلقة

---

## 6️⃣ **ما الفرق بين الأولويات؟**

| الأولوية | اللون | المعنى | التصرف |
|---------|------|-------|-------|
| 🔴 URGENT | أحمر | عاجلة جداً | رد فوري |
| 🟠 HIGH | برتقالي | عالية | رد سريع |
| 🔵 NORMAL | أزرق | عادية (افتراضي) | رد معقول |
| ⚪ LOW | رمادي | منخفضة | يمكن الانتظار |

---

## 7️⃣ **كيف أفلتر الشكاوى؟**

### في لوحة التحكم:

#### البحث:
```
🔍 [______ بحث...______]
```
- ابحث عن: الموضوع، الاسم، البريد، محتوى الرسالة

#### حسب الحالة:
```
الحالة: [جميع ▼]
```
- جميع الحالات
- مفتوحة
- قيد المعالجة
- تم الرد
- مغلقة

#### حسب الأولوية:
```
الأولوية: [جميع ▼]
```
- جميع الأولويات
- عاجلة
- عالية
- عادية
- منخفضة

---

## 8️⃣ **الإحصائيات - ماذا تعني؟**

في الأعلى، ستجد 6 أرقام:

```
📊 15    📨 3    ⏳ 2    ✅ 8    ✔️ 2    🚨 1
إجمالي  مفتوحة  جاري   ردود   مغلقة  عاجلة
```

- **إجمالي (15):** كل الشكاوى
- **مفتوحة (3):** جديدة لم يتم الرد عليها
- **جاري (2):** المسؤول يعمل عليها
- **ردود (8):** تم الرد عليها
- **مغلقة (2):** مغلقة نهائياً
- **عاجلة (1):** الأولوية عاجلة فقط

## 1️⃣1️⃣ **كيف أتأكد أن الشكوى محفوظة صحيح؟**

### الطرق:

#### 1. من الـ Admin Dashboard:
1. افتح لوحة التحكم
2. ابحث عن الشكوى
3. إذا رأيتها = محفوظة بنجاح

#### 2. من قاعدة البيانات:
```sql
SELECT * FROM support_tickets WHERE email = 'user@example.com';
```

#### 3. من API:
```bash
curl -X GET http://localhost:3000/api/v1/support/admin/tickets \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## 1️⃣2️⃣ **من يمكنه الوصول إلى الشكاوى؟**

### المستخدم العادي:
- ✅ يرسل شكوى
- ✅ يرى شكاواه الخاصة فقط
- ❌ لا يرى شكاوى الآخرين

### المسؤول (ADMIN):
- ✅ يرى جميع الشكاوى
- ✅ يرد على أي شكوى
- ✅ يغيّر الحالات
- ✅ يرى الإحصائيات

### الضيف (غير مسجل):
- ✅ يرسل شكوى
- ❌ لا يمكنه متابعة الشكوى لاحقاً
