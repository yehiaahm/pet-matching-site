
### استخدام Prisma في Controllers

#### مثال 1: حفظ مستخدم جديد

```javascript
// في authController.js
export const register = async (req, res) => {
  const { email, password, firstName, lastName } = req.body;
  
  // 1. التحقق من عدم وجود المستخدم
  const existingUser = await prisma.users.findUnique({
    where: { email }
  });
  
  if (existingUser) {
    return res.status(409).json({ error: 'المستخدم موجود بالفعل' });
  }
  
  // 2. حفظ المستخدم الجديد
  const user = await prisma.users.create({
    data: {
      id: randomUUID(),
      email,
      password: hashedPassword,
      firstName,
      lastName,
      createdAt: new Date()
    }
  });
  
  // 3. إرجاع النتيجة
  res.json({ success: true, user });
};
```

#### مثال 2: جلب الحيوانات مع المالك

```javascript
// في petController.js
export const getAllPets = async (req, res) => {
  const pets = await prisma.pets.findMany({
    where: { isActive: true },
    include: {
      users: {  // جلب بيانات المالك أيضاً
        select: {
          id: true,
          firstName: true,
          lastName: true,
          email: true
        }
      }
    },
    orderBy: { createdAt: 'desc' }
  });
  
  res.json({ success: true, data: pets });
};
```

### المميزات الرئيسية

✅ **Type-Safe**: Prisma يوفر types قوية
✅ **Auto-Complete**: IntelliSense كامل في VS Code
✅ **Relations**: جلب البيانات المرتبطة تلقائياً
✅ **Migrations**: إدارة Schema بسهولة

## 3️⃣ تدفق البيانات الكامل

### سيناريو: المستخدم يسجل حيوان جديد

```
1. Frontend (المتصفح)
   └─> المستخدم يملأ نموذج إضافة حيوان
   
2. API Call (src/lib/api.ts)
   └─> petsApi.create(petData)
   └─> POST http://localhost:5000/api/v1/pets
   └─> Headers: { Authorization: "Bearer <token>" }
   └─> Body: { name, species, breed, ... }
   
3. Backend Route (server/routes/petRoutes.js)
   └─> router.post('/', authenticate, createPet)
   
4. Middleware (server/middleware/auth.js)
   └─> التحقق من صحة Token
   └─> إضافة req.user
   
5. Controller (server/controllers/petController.js)
   └─> التحقق من صحة البيانات (Validation)
   └─> استدعاء Prisma
   
6. Prisma (server/config/prisma.js)
   └─> const pet = await prisma.pets.create({ data })
   
7. PostgreSQL Database
   └─> INSERT INTO pets (id, name, species, ...)
   └─> COMMIT
   └─> RETURN inserted_data
   
8. Response Chain
   └─> Prisma ← PostgreSQL
   └─> Controller ← Prisma
   └─> Route ← Controller
   └─> Frontend ← Backend
   
9. UI Update
   └─> تحديث القائمة بالحيوان الجديد
   └─> عرض رسالة نجاح
```

## 5️⃣ معالجة الأخطاء

### في Frontend (`src/lib/api.ts`)

```typescript
try {
  const response = await fetch(url, config);
  
  if (!response.ok) {
    throw {
      status: response.status,
      message: data?.message || 'حدث خطأ'
    };
  }
  
  return data;
} catch (error) {
  console.error('❌ API Error:', error);
  throw error;
}
```

### في Backend (`server/middleware/errorHandler.js`)

```javascript
export const errorHandler = (err, req, res, next) => {
  logger.error('Error:', {
    message: err.message,
    stack: err.stack,
    url: req.url,
    method: req.method
  });
  
  res.status(err.statusCode || 500).json({
    status: 'error',
    message: err.message,
    ...(process.env.NODE_ENV === 'development' && { stack: err.stack })
  });
};
```

## 6️⃣ الأمان (Security)

### Authentication Flow

```
1. المستخدم يسجل الدخول
   └─> POST /api/v1/auth/login
   
2. Backend يتحقق من البيانات
   └─> prisma.users.findUnique({ where: { email } })
   └─> comparePassword(password, user.password)
   
3. إنشاء Tokens
   └─> accessToken (15 دقيقة)
   └─> refreshToken (30 يوم)
   
4. إرسال Tokens
   └─> accessToken في response.body
   └─> refreshToken في httpOnly cookie
   
5. الطلبات اللاحقة
   └─> Frontend يُضيف: Authorization: Bearer <accessToken>
   └─> Backend يتحقق من Token في middleware
```

### Protected Routes

```javascript
// في Backend
import { authenticate } from '../middleware/auth.js';

router.post('/pets', authenticate, createPet);
//                   ^^^^^^^^^^^^ التحقق من Token

// في Frontend
const createPet = async (data) => {
  // api.ts يُضيف Token تلقائياً من localStorage
  return api.post('/pets', data);
};
```

## 8️⃣ الخلاصة

### ✅ كل شيء متصل

1. **Frontend → Backend**: عبر `http://localhost:5000/api/v1`
2. **Backend → Database**: عبر Prisma ORM
3. **API موحد**: `src/lib/api.ts` يدير كل الاتصالات
4. **Schema موحد**: `server/prisma/schema.prisma` يحدد هيكل البيانات
5. **Authentication**: JWT tokens لحماية الطلبات
6. **Error Handling**: معالجة موحدة في كل الطبقات

### ✅ البيانات تُحفظ صح

- كل طلب ناجح يُحفظ في PostgreSQL
- Prisma يضمن Type-Safety
- Transactions تضمن سلامة البيانات
- Indexes تحسن الأداء
- Relations تربط البيانات ببعضها
