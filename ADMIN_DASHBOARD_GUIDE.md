# 🛡️ Admin Dashboard - نظام الإدارة الشامل

## 📋 نظرة عامة

نظام إدارة احترافي متكامل لإدارة المنصة، المستخدمين، والمحتوى. مصمم للتعامل مع متطلبات الشركات الكبرى.

---

## 🎯 المميزات الرئيسية

### 1️⃣ **User Management - إدارة المستخدمين**

#### البيانات المتاحة:
- ✅ قائمة شاملة بجميع المستخدمين
- ✅ بحث متقدم (البريد الإلكتروني، الاسم)
- ✅ تصفية حسب الدور (Admin, Moderator, User, Breeder)
- ✅ تصفية حسب الحالة (نشط، محظور)
- ✅ معلومات التقييم والمطابقات

#### العمليات المتاحة:
- 🔒 **حظر المستخدم**: منع الوصول مع تسجيل السبب
- 🔓 **فك الحظر**: إعادة تفعيل الحساب
- ⚠️ **إضافة تحذير**: يتم الحظر التلقائي بعد 3 تحذيرات
- 👤 **تغيير الدور**: ترقية/تخفيض صلاحيات المستخدم
- 🗑️ **حذف الحساب**: حذف نهائي للحساب (soft delete)

**API Endpoints:**
```
GET    /api/v1/admin/users                    # قائمة المستخدمين
POST   /api/v1/admin/users/ban                # حظر مستخدم
POST   /api/v1/admin/users/unban              # فك حظر
POST   /api/v1/admin/users/warning            # إضافة تحذير
POST   /api/v1/admin/users/change-role        # تغيير الدور
DELETE /api/v1/admin/users/:userId            # حذف الحساب
```

### 2️⃣ **Content Moderation - إدارة المحتوى**

#### أنواع البلاغات:
```
INAPPROPRIATE_CONTENT   - محتوى غير لائق
HARASSMENT              - مضايقات
SPAM                    - رسائل عشوائية
FRAUD                   - احتيال
SCAM                    - نصب
DANGEROUS_BEHAVIOR      - سلوك خطر
ANIMAL_ABUSE            - إساءة للحيوانات
FAKE_PROFILE            - ملف مزيف
OTHER                   - أخرى
```

#### حالات البلاغ:
- `PENDING` - قيد الانتظار
- `REVIEWING` - قيد المراجعة
- `RESOLVED` - تم حلها
- `REJECTED` - مرفوضة
- `APPEALED` - تحت الاستئناف

#### الإجراءات:
- 🚨 **مراجعة البلاغات**: مع وصف المحتوى المُبلغ عنه
- ⚠️ **إرسال تحذير**: للمستخدم المخالف
- 🔴 **حظر المستخدم**: للانتهاكات الخطيرة
- 🗑️ **حذف المحتوى**: إزالة المحتوى المخالف
- 📝 **توثيق الإجراء**: تسجيل سبب الحل

**API Endpoints:**
```
GET    /api/v1/admin/reports                        # قائمة البلاغات
POST   /api/v1/admin/reports/:reportId/resolve      # حل البلاغ
```

### 4️⃣ **System Settings - إعدادات النظام**

#### أوضاع التشغيل:
- 🔧 **وضع الصيانة**: تعطيل الوصول للمنصة
  - إمكانية عرض رسالة مخصصة للمستخدمين
  - مفيد لتحديثات البرامج

#### التحكم في الميزات:
- 👤 تفعيل/تعطيل التسجيل الجديد
- 💞 تفعيل/تعطيل طلبات المطابقة
- 💬 تفعيل/تعطيل الرسائل

#### إعدادات الإشراف:
- ⚠️ الحد الأقصى للتحذيرات قبل الحظر (قابل للتخصيص)

#### المنطقة الخطرة:
- 🔴 مسح الـ Cache
- 🗑️ حذف البلاغات القديمة

**API Endpoints:**
```
GET    /api/v1/admin/settings          # الحصول على الإعدادات
PUT    /api/v1/admin/settings          # تحديث الإعدادات
GET    /api/v1/admin/logs              # سجلات النشاط الإداري
```

## 👥 نظام الأدوار والصلاحيات

### الأدوار المتاحة:

| الدور | الصلاحيات |
|------|----------|
| **SUPER_ADMIN** | كامل الصلاحيات (إدارة الأدوار، الإعدادات) |
| **ADMIN** | إدارة المستخدمين، الحظر، البلاغات |
| **MODERATOR** | مراجعة البلاغات، إضافة تحذيرات |
| **USER** | مستخدم عادي بدون صلاحيات إدارة |
| **BREEDER** | مستخدم مختص في التربية |

### مستويات الوصول:

```typescript
// Super Admin فقط
- تغيير أدوار المستخدمين
- فك حظر المستخدمين
- تحديث إعدادات النظام
- عرض سجلات الأنشطة الإدارية

// Admin و أعلى
- حظر المستخدمين
- إضافة تحذيرات
- مراجعة البلاغات
- عرض الإحصائيات

// Moderator و أعلى
- مراجعة البلاغات فقط
```

## 🔐 الأمان والحماية

### التحقق من الصلاحيات:
```javascript
// في جميع الـ endpoints الإدارية
const requireAdmin = (req, res, next) => {
  if (!['ADMIN', 'MODERATOR', 'SUPER_ADMIN'].includes(req.user.role)) {
    return res.status(403).json({
      success: false,
      message: 'Access denied. Admin privileges required.'
    });
  }
  next();
};

const requireSuperAdmin = (req, res, next) => {
  if (req.user.role !== 'SUPER_ADMIN') {
    return res.status(403).json({
      success: false,
      message: 'Access denied. Super Admin privileges required.'
    });
  }
  next();
};
```

### سجلات التدقيق (Audit Logs):
كل عملية إدارية يتم تسجيلها مع:
- 👤 معرف المسؤول
- 🎯 نوع الإجراء
- 👁️ الهدف (المستخدم/الحيوان/المحتوى)
- ⏰ الوقت والتاريخ
- 🔗 عنوان IP
- 📝 وصف التغييرات

## 🚀 كيفية الاستخدام

### الوصول إلى لوحة التحكم:
```
URL: /admin
الشرط: يجب أن يكون لديك دور Admin أو أعلى
```

### حظر مستخدم:
1. اذهب إلى تبويب **Users**
2. ابحث عن المستخدم
3. اضغط على زر 🔴 (Ban)
4. أدخل السبب
5. أكد الحظر

### حل بلاغ:
1. اذهب إلى تبويب **Moderation**
2. اختر حالة **PENDING**
3. اضغط على **Review**
4. اختر إجراء (Warning, Ban, Delete Content)
5. أضف ملاحظات
6. اضغط **Resolve**

### تفعيل وضع الصيانة:
1. اذهب إلى تبويب **Settings**
2. فعّل **Maintenance Mode**
3. أدخل رسالة للمستخدمين
4. اضغط **Save Settings**

## 📊 أمثلة قاعدة البيانات

### Audit Logs:
```sql
SELECT * FROM audit_logs 
WHERE action = 'BAN_USER' 
AND created_at >= NOW() - INTERVAL '7 days'
ORDER BY created_at DESC;
```

### Reports Statistics:
```sql
SELECT 
  reason,
  status,
  COUNT(*) as count
FROM reports
WHERE created_at >= NOW() - INTERVAL '30 days'
GROUP BY reason, status;
```
